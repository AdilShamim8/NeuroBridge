# Roadmap

This roadmap is public and evolves with community feedback.

## v0.1.0 (released)

Shipped:

- Five built-in cognitive profiles.
- Middleware adaptation pipeline and format adapters.
- REST API server and CLI commands.
- Profile quiz and optional memory backends.
- Security and privacy hardening baseline.
- Demo assets and release documentation.

## v0.2.0 (Q3 2025)

Planned:

- ML-assisted profile detection (opt-in).
- Better profile explainability and controls.
- Expanded integration examples and deployment docs.
- More robust quality gates and benchmark suites.

## v0.3.0 (Q4 2025)

Being considered:

- Team-level policy management for adaptation behavior.
- Multi-language profile variants.
- Telemetry-safe adaptation analytics.
- Extended accessibility adapter ecosystem.

## Ideas We Love But Have Not Committed To

- Browser extension for on-page AI text adaptation.
- IDE-native adaptation overlays during coding.
- Profile-aware summarization benchmarks dataset.
- Structured real-time co-reading mode.

## Ideas We Considered and Decided Against

- Training a custom foundation model from scratch: rejected due to high cost and low integration portability.
- Cloud-only adaptation requirement: rejected to keep local/private deployment possible.
- Forced auto-detection with no user control: rejected to preserve user agency.
