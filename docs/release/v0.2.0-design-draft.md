# v0.2.0 Design Draft (Discussion Seed)

## Problem statement

NeuroBridge v0.1.0 proves adaptation quality and community demand, but release quality gates are fragile in core modules and SDK distribution is incomplete.

## Goals

- Raise engineering confidence through stronger test depth in core runtime paths.
- Stabilize multi-surface distribution (PyPI + npm + docs site).
- Keep product scope tight: quality, reliability, and adoption loops.

## Non-goals

- New cognitive profile taxonomy.
- Breaking API redesign.

## Technical plan

1. Test and reliability track:
   - Add scenario tests for memory persistence, profile switching, and error recovery.
   - Add branch-focused tests for transform pipeline edge conditions.
2. SDK track:
   - Verify package bundles in CI.
   - Add integration tests for OpenAI/Anthropic wrappers with mocked clients.
3. Community telemetry track:
   - Stabilize stats endpoint and graceful fallback behavior.
   - Add documented environment variables for metrics sources.

## Risks

- Coverage improvements can create test maintenance load.
- External API limits can destabilize stats telemetry.
- SDK publishing requires secure credential handling.

## Success criteria

- Coverage >=90% sustained for 3 consecutive CI runs.
- Zero mypy/ruff/black failures on protected branch.
- Published JS SDK preview package and verified install docs.
- Community metrics page updates hourly with no fatal errors.
