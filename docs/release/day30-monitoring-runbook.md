# Day 30 Monitoring and Response Runbook

## Response windows

- Hacker News: respond within 1 hour.
- Reddit: respond within 2 hours.
- Discord joins: welcome each new member personally.
- New contributors: thank by name in `#announcements`.

## Operating checklist

1. Open tabs:
- HN thread
- 4 Reddit threads
- Discord server channels (`#welcome`, `#announcements`, `#help`)
- GitHub notifications/issues/pulls

2. Every 30 minutes:
- Triage comments/questions by urgency and technical complexity.
- Capture recurring feedback into one running note.
- Label actionable feedback as `bug`, `docs`, `dx`, or `research`.

3. Every 2 hours:
- Post a concise status update in Discord.
- Acknowledge top external feedback themes.

4. End-of-day:
- Summarize feedback into:
  - top 5 requests
  - top 3 bugs
  - docs gaps
  - candidate v0.2.0 scope adjustments

## Response style guidelines

- Thank first, explain second.
- Avoid defensiveness; ask clarifying questions.
- Share concrete next steps and timelines where possible.
- Do not over-promise on feature timelines during launch day.
