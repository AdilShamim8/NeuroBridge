# v0.2.0 Scope Lock

Target date: 2026-06-30

## In scope

- Improve profile-detection confidence calibration with real telemetry opt-in.
- Expand test coverage in `core/bridge.py` and `core/memory.py` to sustain >=90% repo target.
- Harden website production analytics + stats reliability.
- Improve JS SDK DX: stricter typings, publishing workflow, and adapter examples.
- Add contributor onboarding automation (issue forms + triage labels).

## Out of scope

- New profile categories beyond current 5 profiles.
- Mobile app clients.
- New UI framework migration.
- Non-Azure deployment automation workstreams.

## Candidate issues to place in milestone

- Coverage hardening: bridge/memory/validators.
- JS SDK package publish and semantic-release wiring.
- Stats page reliability + data freshness checks.
- Performance profiling for high-load adaptation paths.
- Documentation quality automation for docstring examples.

## Closure policy for out-of-scope issues

Close with:

1. Reason issue is out of v0.2.0 scope.
2. Recommended milestone (`v0.3.0` or `research`).
3. Link to design discussion for context.
