# What I Built in 30 Days

In 30 days, I built and launched NeuroBridge: an open-source accessibility layer for AI output.

## The problem

AI output often assumes one reading/processing style. For many neurodivergent users, that creates friction instead of clarity.

## The journey in 5 decisions

1. Build middleware, not a new model.
2. Prioritize deterministic transform quality before advanced ML features.
3. Add profile controls first, then confidence-gated profile inference.
4. Ship across surfaces (package, API, website) to reduce adoption friction.
5. Treat docs, tests, and security checks as product features, not optional polish.

## The numbers

- 30-day build sprint completed
- Python package + CLI + server routes shipped
- Website demo and quiz shipped
- ML profile detector module integrated
- JS/TS SDK scaffold prepared
- Test suite and quality gates in place (coverage improvement still in progress)

## What I learned

- Accessibility quality is mostly implementation discipline.
- Small wording and structure changes can dramatically improve comprehension.
- Community feedback loops are essential for real-world adaptation quality.

## What is next

- Push coverage and reliability targets higher in core runtime modules.
- Improve SDK ergonomics and publishing workflows.
- Expand evidence-backed adaptation evaluation with community input.
