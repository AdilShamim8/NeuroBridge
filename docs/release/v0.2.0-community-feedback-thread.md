# v0.2.0 Community Feedback Thread Draft

Title: Help shape NeuroBridge v0.2.0

We are planning v0.2.0 and want direct feedback from users, maintainers, and accessibility practitioners.

## What we are currently prioritizing

- Reliability and quality gates in core adaptation runtime
- Better JS SDK developer experience
- Community-facing visibility through transparent project metrics

## Questions for the community

1. Which adaptation failures are most disruptive in your workflow?
2. Which SDK integration path should be prioritized first (OpenAI, Anthropic, LangChain, Vercel middleware)?
3. Which docs/examples feel hardest to trust or apply today?
4. What should we measure publicly to prove impact?

## How to contribute

- Comment with use-cases and minimal repro examples.
- Upvote existing pain points rather than creating duplicates.
- Label suggestions as `bug`, `dx`, `docs`, or `research`.

We will consolidate this thread into milestone planning weekly.
